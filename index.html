Machine Learning Advanced

d-RAG-on

My FullStack AI Chatbot built from scratch using a Retrieval-Augmented Generation (RAG) pipeline orchestrated using LangGraph. It retrieves relevant documents from a Pinecone vector store. Then grades its relevance using Groq's llama-3.1-8b-instant model. If the retrieved context is not enough for it, it automatically falls back to live web search and scraping using Tavily. Users can also upload there own documents to extend the knowledge base of the system. The documents are also isolated per user using Pinecone namespaces. The document ingestion pipeline for the .txt and .pdf files are made using a text-splitter and Pinecone multilingual-e5-large inference embedding from Microsoft. The Backend is built using Python(FASTAPI) and the Frontend is made using React deployed on Vercel.

Python JavaScript HTML CSS React Vite LangGraph LangChain Groq Pinecone Tavily Docker

fine-tuning Advanced

CodeJudge

My automated LLM evaluation and fine-tuning framework which benchmarks code generation quality at a large scale. I build a programmatic test suite of 56,000+ test cases, which when ran on the qwen2:1.5b model scored only 35.2% on a simple parsing task, then I used gemma3:12b as a teacher model to generate 83 training examples. Then after that I Fine-tuned qwen2:1.5b via LoRA on Google Colab using Unsloth, improving accuracy from 35.2% → 99.6% — a 64.4 point improvement. This showed a teacher-student knowledge distillation with category-level failure analysis.

Python Ollama LoRA Unsloth mlx-lm Google Colab qwen2:1.5b gemma3:12b
